## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, NavLink inner gap, step-bar gap |
| sm | 8px | Card inner gap, avatar-to-text gap, wizard step segments gap |
| md | 16px | Default horizontal screen padding, form field gap, section padding |
| lg | 24px | Wizard header top padding, modal inner padding |
| xl | 32px | Profile hero padding-top |
| 2xl | 48px | Major section breaks between ProfilePage sections |
| 3xl | 64px | Page-level top inset (onboarding hero) |

Exceptions:

- TabBar sparkle button: 48×48px, margin `-8px` vertical (lifts 8px above bar) — rounded to nearest 4-multiple from prototype's 12px lift
- TabBar pill inner: 8px padding all sides
- Touch targets (all tap buttons): minimum 44px tall — **WCAG 2.1 SC 2.5.5 mandatory minimum — approved exception** (not a multiple of 4; required by accessibility standard)
- Card cover image: `aspect-ratio: 4/5` (no fixed height)
- Stats card grid: 4-column uniform, 8px gap, **8px vertical padding** (inner card content area)
- Wizard progress bar segments: height 4px, **border-radius 4px**, 4px gap between segments

## Interaction Contracts

### Wizard: 4-step Create Flow

Opened by: TabBar sparkle button (routes to `/criar`). The sparkle button must have `aria-label="Criar Prompty"` on the `<button>` element.

| Step | Title | Mandatory fields | Optional fields | Navigation |
|------|-------|-----------------|-----------------|------------|
| 1 — Dados básicos | "Criar Prompty" | `title`, `category` | `complexity_level` (default: simple), `style_tags`, `recommended_model` (expandable) | "Continuar" → step 2 |
| 2 — Prompt | "Criar Prompty" | `beginner_prompt` (non-empty) | — | "Continuar" → step 3 |
| 3 — Imagem | "Criar Prompty" | none | `example_image_url` upload | "Publicar" → publishes; "Pular imagem" ghost link → also publishes |
| 4 — Modo avançado | "Modo avançado (opcional)" | none | `advanced_template`, variable definitions, `inputs_schema` | "Ignorar e publicar" → publishes; "Salvar modo avançado" → saves + publishes |

**Progress bar:** 4 equal-width segments, filled segments use `linear-gradient(90deg, #7C3AED, #22D3EE)`, unfilled use `var(--line)`. Height 4px. Border-radius 4px per segment. Counter "N de 4" monospace at top-right.

**Back navigation:** chevronL icon at top-left. Step 1 → dismisses wizard (go to feed). Steps 2-4 → go to previous step.

**Validation:** "Continuar" is disabled (PrimaryButton disabled state: `var(--line)` background, `var(--text-3)` color) when mandatory field of current step is empty. No inline error messages until submit attempt.

**Publish action:** `status: 'published'` immediately. Toast: "Prompty publicado!" with sparkle icon, `#FF6B4A` color, `+50p` points bubble. Redirect to feed.

### Variation Fork

Triggered by "Criar variação" button on `PromptyDetailPage` (L3 only, visible — not disabled — per LEVL-07).

- Opens wizard at step 1, fields pre-populated from original prompty (title, category, `beginner_prompt`, cover).
- Step 2 prompt textarea pre-filled. Creator edits freely.
- On publish: `parent_id` = original prompty ID. Feed card shows "Baseado em [título original]" as 12px / `var(--text-3)` link below author line.
- Attribution credit style: `font-size: 12px; color: var(--text-3); text-decoration: underline dotted`.

### Advanced Mode Variable Detection

In step 4 textarea (`advanced_template`):

- On every keystroke, detect `{{word}}` patterns via regex `/\{\{([a-zA-Z_][a-zA-Z0-9_]*)\}\}/g`.
- For each unique key found, render a `VariableChip` below the textarea.
- VariableChip contains: key name (monospace, `var(--primary)`), label input (placeholder: "Nome amigável"), type selector (text / enum / number), default value input.
- Live preview section below chips: shows `resolveBeginner(advanced_template, inputs)` output in a read-only `var(--surface-2)` box with mono font.
- Detected count badge: "N variável(is) detectada(s)" — 12px, `var(--text-2)`, shown only when N > 0.
- Highlight `{{key}}` occurrences in textarea: CSS-only not feasible; show count indicator only (do not attempt contentEditable highlight in MVP).

### Stats Grid (Meus Promptys)

Section header: "Meus Promptys" — 12px uppercase / 700 / `var(--text-3)` / letterSpacing 0.6.

Card layout (each card):
- `aspect-ratio: 4/5`, border-radius 12px, `var(--surface)` background, `1px solid var(--line)` border.
- Cover thumbnail: top portion of card (60% height), gradient fallback if no image.
- Bottom portion: title (12px / 700 / lh 1.2 / `var(--text-1)`), then 3 stat rows.
- Inner card content area: **8px vertical padding** (top and bottom of the bottom portion).
- Stat row: icon (16px) + count (14px / 700 / Space Grotesk) + label (12px / `var(--text-3)`). Inline flex, gap 4px.
- Stat icons: `copy` (`#22D3EE`), `bookmark` (`#7C3AED`), `starFill` (`#FFB020`).

Empty state for "Meus Promptys": dashed border card (same pattern as LibraryTab empty state in prototype).

### Ranking Screen (Placeholder)

Route `/ranking`, tab "Ranking" with `starFill` icon.

CONTEXT.md decision: Ranking tab is present but can be a placeholder "Em breve" if no sufficient data.

**Implementation rule:** Render `ComingScreen`-style placeholder (matching prototype `ComingScreen` component):
- 60×60 rounded square `var(--surface-2)` icon container, `sparkle` icon 26px `var(--primary)`.
- Heading "Ranking" — 19px / 700 / Space Grotesk / `var(--text-1)`.
- Body "Os criadores que mais contribuíram aparecem aqui. Em breve!" — 14px / `var(--text-2)` / lh 1.5 / maxWidth 280 centered.
- When real ranking data exists (Phase 3+ post-MVP), replace with full leaderboard.

## Animation Contract

All animations use existing keyframes from `src/index.css`. No new keyframes are introduced.

| Keyframe | Duration / Easing | Usage in Phase 3 |
