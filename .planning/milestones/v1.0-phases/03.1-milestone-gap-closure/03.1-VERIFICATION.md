**Expected:** Profile shows the incremented points value immediately after the copy, same session.
**Why human:** The gamification loop involves a SQL trigger (`record_copy` RPC) followed by a re-fetch. The unit test covers the refetchProfile call is made, but not that the trigger fires and the DB value actually changes.
