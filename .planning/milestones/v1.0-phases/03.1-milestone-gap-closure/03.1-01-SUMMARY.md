## What Was Built

### Gap 1: Gamification loop (LEVL-01/02/03)

`refetchProfile()` added to the `AuthStore` interface and implementation. The action reads `useAuthStore.getState().user`, fetches the `profiles` row from Supabase, and calls `setProfile()` on success. On error, it silently returns without mutating state.

Both `useCopy.copy()` (after the `record_copy` RPC block) and `useTest.submit()` (on the success path) call `void useAuthStore.getState().refetchProfile()` — fire-and-forget, never blocking user feedback.

### Gap 2: Route-level level gating (LEVL-07/CUR-01/CUR-02/CREAT-01)

`PrivateRoute` extended with `minLevel?: 'L1'|'L2'|'L3'|'L4'|'L5'` and `routeLabel?: string` props. Level comparison uses `LEVEL_ORDER.indexOf()` (same pattern as TabBar). Level-insufficient users get `<Navigate to="/" state={{ levelGate: routeLabel }}>`.

In `App.tsx`, `/saved` and `/search` are wrapped in `<PrivateRoute minLevel="L2" routeLabel="Salvos e Buscar">` and `/criar` and `/ranking` in `<PrivateRoute minLevel="L3" routeLabel="Criar Promptys e Ranking">`.

`FeedPage` reads `useLocation().state.levelGate` and renders a lock Toast when present.

### Gap 3: MODR-03 useSaved status filter

Both parallel Supabase queries in `useSaved` now chain `.eq('promptys.status', 'published')` after `.order()`. PostgREST v12 supports filtering on embedded resource columns via dot notation.
