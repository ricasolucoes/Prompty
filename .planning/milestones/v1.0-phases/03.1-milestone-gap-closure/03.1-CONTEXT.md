# Phase 3.1: Milestone Gap Closure - Context

**Gathered:** 2026-05-13
**Status:** Ready for planning
**Source:** v1.0 Milestone Audit (`.planning/v1.0-MILESTONE-AUDIT.md`)

<domain>
## Phase Boundary

This phase exists to close 5 cross-phase integration gaps identified by the v1.0 milestone audit. Each phase (1, 2, 3) passed its own VERIFICATION.md, but the milestone audit caught cross-cutting issues that no single phase was responsible for. Phase 3.1 is a focused, narrow gap-closure phase — no new features, no scope expansion.

The 5 gaps to close:

1. **Gamification loop stale within-session** — SQL triggers update profiles.points correctly, but the frontend AuthStore.profile.points is never refreshed after useCopy/useTest. Affects LEVL-01, LEVL-02, LEVL-03 (LEVL-04 fires correctly once points are current).
2. **Route-level gating missing** — PrivateRoute exists but is never used. /saved, /search, /criar, /ranking are reachable by direct URL bypassing TabBar enforcement. Affects LEVL-07, CUR-01, CUR-02, CREAT-01.
3. **useSaved misses MODR-03 status filter** — moderated promptys can still appear in Salvos/Avaliações/Resultados tabs because useSaved doesn't filter promptys.status on the join. Affects MODR-03.
4. **ProfilePage progress nudge hardcoded** — body text always says "Buscar e Salvos" regardless of current level. For L2→L3, the unlock is "Criar Promptys", not "Buscar e Salvos". Affects LEVL-05.
5. **VariableChip orphaned from Step 2** — WizardStep2Prompt accepts `{{var}}` silently; only Step 4 detects variables. Creators publishing without reaching Step 4 ship empty inputs_schema. Affects CREAT-02, CREAT-05.

</domain>

<decisions>
## Implementation Decisions

### Gap 1: Gamification refetch (LEVL-01, LEVL-02, LEVL-03)

- **Approach:** Add `refetchProfile()` call to AuthStore (fetches `profiles` row by `auth.uid()` and updates store). Call from useCopy.copy() success and useTest.submit() success. Fire-and-forget (no await blocking the UI).
- **Not chosen:** Supabase Realtime subscription on profiles. Reason: adds connection overhead, complicates teardown, overkill for a 2-call solution.
- **Not chosen:** Polling. Reason: wasteful for an event-driven system.
- **Test:** After mocked useCopy.copy() resolves, the AuthStore mock receives a refetch call. RTL test asserts the call happens.

### Gap 2: Route-level guards (LEVL-07, CUR-01, CUR-02, CREAT-01)

- **Approach:** Use existing `src/components/layout/PrivateRoute.tsx`. Wrap /saved (minLevel=L2), /search (minLevel=L2), /criar (minLevel=L3), /ranking (minLevel=L3) in App.tsx.
- **Redirect behavior:** Unauthenticated users → /login (PrivateRoute's existing fallback). Users below required level → /feed with a toast "Promptys precisa de mais experiência para liberar X" (or similar — copy at Claude's discretion).
- **Test:** RTL test asserts that an L1 user navigating to /saved gets redirected; an L2 user gets the page.

### Gap 3: useSaved MODR-03 filter

- **Approach:** Add `.eq('promptys.status', 'published')` on both queries in useSaved (the prompty_saves join and the prompty_tests join). Alternative if PostgREST syntax doesn't allow filtering on joined columns: post-fetch filter the joined `promptys` field.
- **Test:** Mock returns saves where one prompty has status='flagged'. Hook filters it out.

### Gap 4: ProfilePage nudge text

- **Approach:** Generate body text from `next.id` using a lookup map: L2 → "Buscar promptys e salvar para depois", L3 → "Criar seus próprios promptys", L4 → "Ranking e badges", L5 → "Modo avançado completo". The current text is fine for L1→L2.
- **Location:** src/pages/ProfilePage.tsx around line 117.
- **Test:** Render ProfilePage with profile.points=60 (L2). Assert text contains "Criar".

### Gap 5: Step 2 variable detection (CREAT-02, CREAT-05)

- **Approach:** In WizardStep2Prompt, after the textarea, render `extractVariables(beginnerPrompt)` as a small inline message + chip preview. If variables detected, show "Detectamos {{X}}, {{Y}} — para configurar tipos e defaults, vá para o modo avançado (Step 4)". This is a notice, not a chip editor — Step 4 stays the canonical editor.
- **Reason:** Surfaces detection without duplicating the schema management logic. Creators get visible feedback that variables exist.
- **Test:** Type a template with {{name}} in step 2 textarea. Assert "Detectamos {{name}}" hint renders.

### Claude's Discretion

- Exact toast copy for level-gate redirects (Gap 2)
- Exact nudge body strings for each level (Gap 4)
- Exact hint wording for Step 2 variable detection (Gap 5)
- Whether refetchProfile lives in AuthStore or as a new hook (Gap 1) — recommend in AuthStore for consistency with existing setUser/setProfile
- Whether the Step 2 hint links to Step 4 with a "Configurar agora" button or is informational only — informational is simpler

</decisions>
