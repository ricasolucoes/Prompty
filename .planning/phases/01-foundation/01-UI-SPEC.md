## Required CSS Animations (globals.css)

```css
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(6px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pop {
  0%   { transform: scale(0.6); opacity: 0; }
  60%  { transform: scale(1.15); opacity: 1; }
  100% { transform: scale(1); }
}

@keyframes slideUp {
  from { transform: translateY(100%); }
  to   { transform: translateY(0); }
}

@keyframes shimmer {
  0%   { transform: translateX(-100%); }
  100% { transform: translateX(100%); }
}

.screen { animation: fadeIn 0.25s cubic-bezier(0.2, 0.8, 0.2, 1); }
```

Usage:
- `fadeIn` — card skeleton → content transition, overlay appear
- `pop` — LevelUpModal entrance: `cubic-bezier(.2, 1.4, .4, 1)` (bouncy spring)
- `slideUp` — bottom sheet / RateSheet entrance: `cubic-bezier(.2, .8, .2, 1)`
- `shimmer` — skeleton loading pulse (via overflow:hidden + absolute gradient)
- `screenFade (.screen)` — screen mount transition: `.25s cubic-bezier(.2, .8, .2, 1)`

> Source: CONTEXT.md § Animações CSS obrigatórias, Promptys v2.html lines 50–54.

## Typography

Exactly 4 size roles. Exactly 3 weights: 400 (regular), 600 (labels/medium), 700 (headings/bold).

| Role | Size | Weight | Font | Line Height | Usage |
|------|------|--------|------|-------------|-------|
| Body | 13.5px | 400 | Inter | 1.5 | Prompt preview text in feed card, body copy, post-copy banner body, toast text, author subline |
| Label | 12px | 600 | Inter | 1.2 | Author name subline (time), chip labels, section uppercase labels, reaction count, copies count, like count |
| Heading | 19px | 700 | Space Grotesk | 1.25 | Feed card title (`<h2>`), letter-spacing: -0.4px |
| Display | 22px | 700 | Space Grotesk | 1.1 | Profile name, header logo text, letter-spacing: -0.5px |

Variants within roles (not new roles):
- **Body-Bold** — 13.5px / weight 700 / Inter: action button labels (Curtir, Copiar prompt), author full name in card header, post-copy banner headline, "Ver mais" inline button
- **Label-Caps** — 12px / weight 600 / Inter / uppercase / letter-spacing 0.6px: section labels ("PROMPT", "SEUS ÚLTIMOS USADOS", "COMO FUNCIONA"), header level badge text (letter-spacing 0.4px)

Prompt text (special role):
- 13.5px / 400 / Inter (NOT JetBrains Mono in L1 feed card) — the body text display
- JetBrains Mono reserved for: code blocks, version numbers, monospace counters (not shown in L1)

> Source: Promptys v2.html FeedCard inline styles; docs/design/design-system.md § Tipografia.
> Note: prototype uses 13.5px for prompt body, not 13px. Use exact value.
> Weights 500 and 800 are NOT declared — removed. Where 500 was used: use 600. Where 800
> was used (Onboarding title, LevelUp level name, "NOVO NÍVEL" label): use 700.

## Visual Hierarchy

**Primary focal point — Feed screen:** the FeedCard as a unit, anchored visually by the "Copiar prompt" primary CTA button (gradient violet, full-width, at card bottom). Every other element in the card exists to establish enough context for the user to tap that button with confidence.

**Accent (`--primary`) reserved for:**

- 60%: `--bg` fills the scrollable viewport and between-card gaps

## Interaction Contracts

### Copy Flow (primary loop)
```
Feed card: [Copiar prompt]
  → navigator.clipboard.writeText(resolvedPrompt)
  → Button text becomes "Copiado!" + icon changes to check (color: #34D399)
  → Post-copy banner appears inside the card (fadeIn .25s)
  → Toast: "Prompt copiado" (icon: copy, color: #34D399) — auto-dismiss 2400ms
  → User taps "Avaliar este prompt (+5p)"
    → RateSheet slides up (slideUp .25s)
    → User selects 1–5 stars
    → Optional: attaches image
    → [Enviar (+5p)] submits (insert into prompty_tests → trigger → +5p)
    → RateSheet closes
    → Toast: "Avaliação enviada" (icon: check, color: #34D399, points: "+5p")
    → Banner replaced by green confirmation row
```

### "Ver mais" behavior (Claude's Discretion)
Expand inline — do NOT open a modal. When tapped:
- Remove `-webkit-line-clamp` constraint
- Text expands in place within the card
- No collapse button needed (low priority)
- Smooth height transition: `max-height` CSS transition or layout reflow accepted

### Like button
- Toggle state immediately (optimistic update)
- Icon animates: `scale(1.3)` then `scale(1)` in 150ms on activation
- Color transitions: `var(--text-2)` → `var(--like)` on active
- Authenticated only: if anonymous user taps, redirect to `/login` (or show auth prompt — per CONTEXT.md, no modal)

### Tab bar navigation
- L1: 2 tabs — Feed + Perfil. No other tabs exist or are hinted at.
- Active tab icon strokeWidth increases: 1.8 → 2.2
- Tab bar never shows lock icons, disabled states, or coming-soon states

### Screen transitions
- All screen mounts: `animation: fadeIn .25s cubic-bezier(.2, .8, .2, 1)` via `.screen` class
- Back navigation: no exit animation (instant remove), new screen fades in

### LevelUp detection
- Triggered when `levelOf(profile.points)` changes to a higher level
- Modal shown once per level threshold — persisted to prevent showing again after dismiss
- Tab bar animates new icons appearing after modal dismiss (not before)

### Skeleton loading (Claude's Discretion)
- Show 2 skeleton cards immediately on mount
- Each skeleton: same dimensions as FeedCard
- Shimmer: horizontal gradient sweep, duration 1.4s, repeat infinite
- Skeletons fade out and real cards fade in on data arrival

## Level Data Constants

```typescript
// src/lib/constants/levels.ts
export const LEVELS = [
  { id: "L1", name: "Explorador",  min: 0,    color: "#22D3EE", emoji: "🌱", desc: "Você está começando — copie, gere, avalie." },
  { id: "L2", name: "Curador",     min: 50,   color: "#7C3AED", emoji: "✨", desc: "Buscar, salvar, seguir criadores." },
  { id: "L3", name: "Criador",     min: 250,  color: "#FF6B4A", emoji: "🎨", desc: "Criar Promptys, remixar e disputar o ranking." },
  { id: "L4", name: "Mestre",      min: 1000, color: "#FFB020", emoji: "👑", desc: "Reconhecido pela comunidade." },
  { id: "L5", name: "Lendário",    min: 5000, color: "#EC4899", emoji: "🔮", desc: "Topo do topo." },
] as const;

export function levelOf(points: number) {
  let cur = LEVELS[0];
  for (const l of LEVELS) if (points >= l.min) cur = l;
  return cur;
}
```
