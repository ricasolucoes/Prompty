  { to: '/create',  icon: 'sparkle',  label: 'Criar',  minLevel: 'L3' }
  { to: '/ranking', icon: 'flame',    label: 'Ranking',minLevel: 'L3' }
  { to: '/profile', icon: 'user',     label: 'Perfil', minLevel: 'L1' }

L1 still sees exactly 2 tabs (Feed + Perfil). L2 sees 4 (Feed + Salvos + Buscar + Perfil). L3 sees 6.

**TabBar test update required:** `TabBar.test.tsx` currently asserts L2 = 3 tabs. Must be updated to assert L2 = 4 tabs in the same plan wave that modifies `TabBar.tsx`.

### FilterChipBar — single-select tap behavior

```
User taps chip:
  if chip is already active → deactivate (set activeChip = null)
  if chip is inactive → activate (set activeChip = chip.value)
  → triggers useSearch/useSaved refetch with updated filter params
  → results update immediately (no loading spinner for filter change —
    optimistic render: show previous results while new page loads,
    then replace)
```

### Search input — debounced query flow

```
User types in search field:
  → setQuery(value) [immediate — for controlled input display]
  → clearTimeout(timerRef.current)
  → timerRef.current = setTimeout(() => setDebouncedQuery(value), 300)

debouncedQuery change:
  → useSearch re-runs (queryKey updates → TanStack Query refetch)
  → if empty + no filters → show "idle" empty state
  → if loading → show 2 SkeletonCards
  → if results → render FeedCards
  → if 0 results → show "Nenhum resultado" empty state
```

Clear button (Icon `x`) on input:
```
User taps clear:
  → setQuery('')
  → setDebouncedQuery('') immediately (no debounce on clear)
  → returns to idle empty state
```

### "..." menu flow

```
User taps "..." button on PromptyDetailPage (L2+ only):
  → OptionsSheet opens (slideUp .25s)
  → Two options visible: "Sugerir categoria" | "Denunciar"

User taps "Sugerir categoria":
  → OptionsSheet closes
  → CategorySuggestSheet opens (slideUp .25s)

User taps "Denunciar":
  → OptionsSheet closes
  → ReportSheet opens (slideUp .25s)
```

### Report submission flow

```
User fills reason dropdown (required) + optional notes:
  → "Denunciar" button enables
  → User taps "Denunciar"
    → setBusy(true)
    → useReport.submit({ prompty_id, reason, type: 'report', notes? })
    → on success: ReportSheet closes, Toast "Denúncia enviada"
    → on error: err state shown, sheet stays open
```

### Category suggestion flow

```
User selects category from dropdown:
  → "Enviar sugestão" button enables
  → User taps "Enviar sugestão"
    → useReport.submit({ prompty_id, reason: selectedCategory, type: 'category_suggestion', notes? })
    → on success: CategorySuggestSheet closes, Toast "Sugestão enviada"
    → on error: err state shown
```

### Community results — load behavior

```
PromptyDetailPage mounts:
  → if isL2: useCommunityResults(prompty.id) fetches prompty_tests WHERE
      prompty_id = id AND image_url IS NOT NULL ORDER BY created_at DESC
  → if results.length === 0: section is not rendered (absent, not empty state)
  → if results.length > 0: section renders with shimmer tiles → loaded images
  → Each tile tap: FullImageModal opens (fadeIn .2s)
  → FullImageModal backdrop/X tap: modal closes
```
