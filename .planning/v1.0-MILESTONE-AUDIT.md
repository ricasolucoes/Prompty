## Cross-Phase Integration Gaps

### Gap 1: Gamification loop broken in same session (HIGH)

**Affects:** LEVL-01, LEVL-02, LEVL-03, LEVL-04

After `useCopy.copy()` calls `record_copy` RPC or `useTest.submit()` inserts `prompty_tests`, SQL triggers correctly update `profiles.points` server-side. But the frontend `AuthStore.profile.points` is only populated at:
- App startup (main.tsx)
- `onAuthStateChange` events (login/logout)
- `useProfile.update()` (profile edits)

No hook refetches profile after a points-awarding action. Consequences:
- `levelOf(profile.points)` returns stale level in TabBar, PromptyDetailPage gating, MyPromptysGrid
- ProfilePage ProgressBar shows stale points
- LevelUpModal never fires in the session where a level-up actually happened

**Fix shape:** Add `await refetchProfile()` after `useCopy.copy()` and `useTest.submit()` succeed, OR subscribe to `profiles` table via Supabase Realtime on `auth.uid()`.

### Gap 2: Routes unprotected by level gating (HIGH)

**Affects:** LEVL-07, CUR-01, CUR-02, CREAT-01

`src/components/layout/PrivateRoute.tsx` is defined but never imported anywhere. `App.tsx` mounts `/saved`, `/search`, `/criar`, `/ranking` as plain `<Route>` children. TabBar correctly hides locked tabs, but direct URL access (typed, shared link, browser history) bypasses enforcement.

**Fix shape:** Wrap each L2+/L3+ route in `<PrivateRoute minLevel="L2">` or similar — the component exists; just needs to be used.

### Gap 3: useSaved doesn't filter moderated promptys (MEDIUM)

**Affects:** MODR-03

`src/hooks/useSaved.ts` joins `promptys` for cover/title display but doesn't add `.eq('promptys.status','published')`. A moderated prompty can still appear in Salvos/Avaliações/Resultados tabs.

**Fix shape:** Add status filter to the join or post-fetch filter.

### Gap 4: ProfilePage nudge copy hardcoded (MEDIUM)

**Affects:** LEVL-05

`src/pages/ProfilePage.tsx` line 117: nudge body always says "Use mais alguns Promptys para destravar Buscar e Salvos." For an L2 user approaching L3, this is incorrect — they're unlocking Criar Promptys.

**Fix shape:** Generate body text from `next.name` and per-level unlock list.

### Gap 5: VariableChip orphaned from Step 2 (LOW)

**Affects:** CREAT-02, CREAT-05

`WizardStep2Prompt` accepts `{{var}}` syntax in beginner_prompt without any visual feedback or schema population. Creators publishing without reaching Step 4 ship empty `inputs_schema`, causing `resolveBeginner()` to render `{{var}}` as literal text to consumers.

**Fix shape:** Either (a) reuse VariableChip in Step 2 for live detection, or (b) document at publish time that variables only work via Step 4 advanced mode.
